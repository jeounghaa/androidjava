1. 형변환이 무엇인가?
>> 타입을 변환하는 것
2. 자동형변환은 무엇인가? 언제? 
>> 크기가 작은 데이터 타입을 큰 데이터 타입으로 변환하는 경우
   byte를 int로 변환
3. 강제형변환은 무엇인가? 언제?
>> 크기가 큰 데이터 타입을 작은 데이터 타입으로 변환하는 경우
   int를 byte로 변환
4. 강제형변환을 할 수 없을 때는?
>> int a = 128;
   char b;일 경우에 
   int인 a는 char인 b의 타입에 들어갈 수 없다.
5. 클래스다이어그램을 그리세요.
>> Car
	↑
  Truck
6. 코드와 비교하면서 자동형변환과 강제형변환 확인해보세요.
>> byte a = 10; int b = a;   -> 자동형변환
   int a = 20; byte b = (int) a;   -> 강제형변환
7. 클래스간 형변환은 언제 쓸 수 있나요?
>> 클래스간 상속관계일 경우에만 사용할 수 있다.
8. 클래스간 자동 형변환을 부르는 영어는?
>> up casting
9. 형변환을 영어로?
>> casting
10. 클래스간 강제 형변환을 부르는 영어는?
>> down casting
11. 강제 형변환의 문법은?
>> 데이터타입 변수명 = (데이터타입) 값?
12. ArrayList를 왜 썼나요?
>> 다양한 타입들의 값들을 유동적으로 사용하기 위해서
13. ArrayList에 값을 넣을 때 내부적으로 일어나는 일은?
>> ArrayList는 여러 타입들의 값이 Object로 업캐스팅해서 들어간다.
14. ArrayList에서 값을 꺼내올 때 내부적으로 일어나는 일은?
>> Object인 값을 다운캐스팅해서 값을 꺼내온다.
15. Wrapper Class(포장 클래스)란?
>> 기본형변수와 참조형변수를 서로 변환하고자할 때 사용하는 클래스
16. boxing은?
>> 기본형변수를 참조형변수로 형변환하는 것
17. unboxing은?
>> 참조형변수를 기본형변수로 형변환하는 것
